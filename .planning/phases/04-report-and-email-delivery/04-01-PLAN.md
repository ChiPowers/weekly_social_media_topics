**Module-level Jinja2 environment** (using absolute path to avoid cwd bugs):
```python
import os
from jinja2 import Environment, FileSystemLoader, select_autoescape

_TEMPLATE_DIR = os.path.join(os.path.dirname(__file__), "templates")

_jinja_env = Environment(
    loader=FileSystemLoader(_TEMPLATE_DIR),
    autoescape=select_autoescape(enabled_extensions=("html",)),
)
```

**`_format_date(run_date: str) -> str`** — module-level helper:
- Input: ISO date string e.g. "2026-03-03"
- Output: "March 3, 2026"
- Implementation: `datetime.strptime(run_date, "%Y-%m-%d").strftime("%B %-d, %Y")`
- Note: `%-d` removes leading zero on Linux/macOS (safe for this project's targets)

**`EmailDeliverer` class:**

`__init__(self, config: Config)`:
- Store config reference
- Set `resend.api_key = config.resend_api_key` ONCE here — do NOT set it inside send methods (avoids test pollution; per research pitfall 3)
- Initialize `self._logger = logging.getLogger(__name__)`

`render_report(self, report: IdeaReport) -> tuple[str, str]`:
- Returns `(subject, html_body)`
- Subject: `f"{len(report.ideas)} content idea{'s' if len(report.ideas) != 1 else ''} for this week"`
- HTML: render report.html template with `ideas=report.ideas, header_date=_format_date(report.run_date)`

`render_plain_text(self, report: IdeaReport) -> str`:
- Build plain text directly from IdeaReport fields (no template — just string construction)
- For each idea: numbered line, angle indented, bullets indented with "- ", rationale indented, optional "Best for: X" / "Format: Y" lines, blank line separator
- Return `"\n".join(lines)`

`send_report(self, report: IdeaReport) -> None`:
- Call `render_report()` to get subject and html
- Call `render_plain_text()` to get text
- Call `resend.Emails.send()` with params: from=config.email_from, to=[config.email_recipient], subject, html, text
- On success: `self._logger.info("Email sent to %s at %s", config.email_recipient, datetime.now().isoformat())`
- Raise `resend.exceptions.ResendError` subclasses — do NOT catch them here; let them propagate to pipeline.py

`send_error(self, reason: str) -> None`:
- Build subject: `f"Content agent error — {_format_date(datetime.now().strftime('%Y-%m-%d'))}"`
- Render error.html template with `reason=reason`
- Plain text: just the reason string
- Call `resend.Emails.send()` with same from/to as above, error subject, html, and text
- Wrap the ENTIRE send_error body in `try/except resend.exceptions.ResendError` — log and swallow on failure (per research pitfall 4: error email delivery can itself fail; must not mask original exception)

**Imports needed:** `import logging`, `import resend`, `import resend.exceptions`, `from datetime import datetime`, `from agent.config import Config`, `from agent.models import IdeaReport`
